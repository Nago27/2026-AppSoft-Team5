# 응용소프트웨어실습(금요일) 5조 프로젝트 제안서

#### **프로젝트명**: RPG 요소를 결합한 할 일 관리(To-do List) 프로그램 
----

## 1. 관련조사 분석 및 목표 선정 배경
#### 1-1. 주제 선정 배경
기존의 일반적인 To-do List 서비스들은 할 일을 기록하고 완료 여부를 체크하는 기능이 충분하지만, 사용자가 지속적으로 사용하도록 유도하는 동기부여 요소가 부족하다는 한계점이 존재했습니다. 특히 공부, 자기관리와 같은 반복적인 활동의 경우, 단순의 체크 기능만으로 해당 서비스에서 성취감이나 재미를 주기엔 부족하다고 생각했습니다.
<br> 따라서, 기존 To-do List의 서비스는 유지하면서 사용자가 할 일을 수행하는 과정에서 보상과 성장의 재미를 함께 경험할 수 있도록 **RPG 요소를 결합한 서비스**를 기획하게 되었습니다. 단순히 일정만 관리하는 것이 아나라, 사용자의 행동이 캐릭터 성장과 연결되도록 하여 To-do List의 사용 동기를 높이고자 합니다.

#### 1-2. 기존 서비스 분석 및 차별화 방향
할 일 관리에 게이미피케이션(Gamification)을 도입한 실제 사례를 조사하던 중, **Habitica**라는 앱을 접하게 되었습니다. 캐릭터 육성과 할 일 목록을 연동한 점이 저희 주제와 유사하여, 해당 서비스를 벤치마킹하여 주요 기능을 분석하고 저희만의 차별화된 요소를 도출해 보았습니다.

- **국내 사용자 접근성 개선**: 현재 서비스는 한국어를 지원하지 않아 국내 사용자의 진입 장벽이 높습니다. 이를 해결하고자 한국어 기반 UI로 구현하여 접근성을 높이고자 합니다.
- **시각적 효과(애니메이션)**: 할 일 완료 시의 성취감을 높여줄  애니메이션 연출이 부족해 정적인 느낌이 있었습니다. 따라서, 여러 시각 효과를 추가하여 게임적 재미를 주고자 합니다.
- **가이드 시스템**: 복잡한 RPG 요소에 비해 도움말 서비스가 부족하여 진입 장벽이 높다고 느꼈습니다. 단계별 도움말 시스템을 설계하여 사용자 편의성을 제공하고자 합니다.
- **확률성 보상 요소 도입**: 기존 서비스의 단조로운 보상 구조를 개선하기 위해 **확률성 보상 시스템**을 도입하여, 사용자의 지속적인 동기 부여와 흥미를 유발하고자 합니다.

## 2. 최종 설계 목표
프로젝트의 최종 목표는 To-do List 기능을 중심으로 RPG 요소를 결합한 웹 서비스를 구현하는 것입니다. 팀프로젝트의 전체 일정과 완성도를 고려하여, 모든 아이디어를 한 번에 구현하기보다는 핵심 기능을 우선적으로 완성하는 방향으로 조율하였습니다.

#### 2-1. 주요 구현 기능
##### 1) To-do List 
- 사용자가 할 일을 등록, 수정, 삭제, 완료할 수 잇는 기본적인 To-do List 기능 구현
- 할 일 완료시 보상 시스템: 
    - 획득한 코인을 활용해 확률형 뽑기 사용 가능
    - 카테고리 별 캐릭터 스텟이 올라가는 비율을 다르게 설정
    - 특정 조건 만족시 업적 달성 시스템

##### 2) 사용자 캐릭터
- 사용자의 To-do 수행결과를 반영한 개인 캐릭터
- 캐릭터 스텟: 레벨 및 경험치, 체력, 코인

##### 3) 시각 효과(Effect)
- 할일 완료, 보상 획득, 캐릭터 레벨업 등 주요 상황에서 시각적인 효과 제공

##### 4) 던전
- 일반적인 전투 방식이 아닌, To-do List 기반 클리어 방식
- 기한이 임박하거나 중요한 일에 대해서 To-do를 만들 수 있는 방식으로 적용

#### 2-2. 적용 기술 스택
본 프로젝트는 다음과 같은 C# 언어를 중심으로 구현하고자 합니다.
##### 1) C# 핵심 기능 및 객체지향 설계 활용
To-do, 사용자 계정, 캐릭터 등의 데이터를 체계적으로 관리하기 위해 C#의 Class를 활용한 객체지향 설계를 적용하고자 합니다. 

##### 2) `ASP.NET` Core Blazor
웹 기반 서비스 UI 및 로직 구현을 위해 `ASP.NET` Core Blazor 프레임워크를 선택했습니다. Blazor를 통해 C# 중심으로 화면과 기능을 구성할 수 있으며, 사용자 입력에 따라 동적으로 화면을 갱신하는 구조를 구현이 가능합니다.

##### 3) SQLite
데이터 저장을 위해 SQLite 로컬 DB를 사용할 계획입니다. 회원가입 정보, 할 일 목록, 캐릭터 정보 등 서비스 이용에 필요한 데이터를 저장하고 관리하는 데 활용하고자 합니다.
